# AI Dehazing Project - Codebase Audit Report

## Project Summary
This codebase implements an AI-driven real-time video dehazing system with a React Native (Expo) frontend, a Node.js Express backend, and Python-based AI processing.

## 1. Directory Structure & Entry Points
- **Root**: Expo project configuration (`app.json`, `package.json`).
- **`app/`**: Application screens (Index, Capture, Processing, Evaluation).
- **`dehazing-backend/`**: Node.js backend with WebSocket and GridFS support.
- **`scripts/`**: Python AI models (`mlkd_net.py`, `aod_net.py`, `yolo_eval.py`).

## 2. Component Breakdown

### Frontend (Expo)
- **Navigation**: `expo-router` (Stack-based).
- **Real-time Capture**: Managed in `app/capture.js` via `react-native-vision-camera`.
- **Communication**: Custom `WebSocketService` in `app/_layout.js`.
- **Authentication**: `app/services/authservice.js` (JWT & MFA supported).

### Backend (Node.js)
- **API**: Express.js with routes for Auth, Processing, and Real-time data.
- **WebSocket**: Primary hub for frame-by-frame processing in `websocketService.js`.
- **Storage**: MongoDB GridFS for high-volume image data.
- **AI Bridge**: `aiService.js` spawns Python processes for dehazing.

### AI Processing (Python)
- **MLKD-Net**: Simulated in `scripts/mlkd_net.py` using OpenCV (CLAHE + Guided Filter).
- **AOD-Net**: Lightweight alternative for faster processing.
- **YOLO**: Object detection evaluation implemented for performance benchmarking.

## 3. Key Findings
- **Real-time Performance**: Achieved through 320x240 resolution scaling and WebSocket bidirectional streaming.
- **Dynamic Mode Switching**: Backend automatically switches from Cloud to Local mode if latency spikes.
- **Security**: Implements MFA-based login and JWT-protected API routes.
